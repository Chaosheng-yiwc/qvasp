Running Ways
---------------------      
There are 2 ways to run qvasp under command line mode:

1. Just type ``qvasp -pbe C H`` in the terminal to generate POTCAR;
2. ``echo -e "\n \n" | qvasp -bandd``;

Run qvasp in Batch Mode
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

if one wants to run qvasp in batch mode, for example, to generate POTCAR file in several sub-olders with POSCAR:

.. code-block:: none

   for i in `ls -l |grep -a "^-"|awk '{print $NF}'`
   do 
       cd $i
	   elements=`cat POSCAR|sed -n 6p`
       qvasp -pbe $elements
	   echo $i "done"
       cd ..
   done

if one wants to extract the specified physical quantity, for example, to read band gap in several sub-olders:

.. code-block:: none

   for i in `ls -l |grep -a "^-"|awk '{print $NF}'`
   do 
       cd $i
       bandgap=`echo -e "\n \n" | qvasp -bandd | sed -n '7p' | awk '{print $7}'`
	   echo "Band gap of "  $i  “ is ” $bandgap
       cd ..
   done

Generate Input Files
-----------------------

To perform a firt principle calculation using VASP, usually one needs 4 files, INCAR,POSCAR, POTCAR and KPOINTS.

1. INCAR contains all keywords and tells VASP what to calculate;

2. POSCAR contains lattice parameters, atomic coordinates information,and atomic velocity information (for MD);

3. KPOINTS, contains K-points information in the reciprocal space, at which the wave function integrates to obtain the charge density.

4. POTCAR is a pseudo potential file, which is USPP or PAW type;

qvasp provide flexible toolkit to generate the input files for VASP.

Generate POSCAR
~~~~~~~~~~~~~~~~~~

Usually, we recommand generate POSCAR firstly.

.. code-block:: none

   > qvasp -c2p C.cif

or 

.. code-block:: none

   > qvasp -res2vasp C.res

If the POSCAR belongs to solid surface model, and need to fix the atomic potsions in botom layers. Further use:

.. code-block:: none

   > qvasp -fix POSCAR

Then the fix scale need to be inputted, one should note that the vacuum layer should along z direction.

.. code-block:: none

   >cat POSCAR
   Transfer by qvasp
    1.0
    2.522100    0.000000    0.000000    
    1.261050    2.184203    0.000000    
    1.261050    0.728068    2.059286    
  C   
  2  
  Direct
  0.000000  0.000000  0.000000  
  0.250000  0.250000  0.250000


Generate KPOINTS
~~~~~~~~~~~~~~~~~~~~

The generation of KPOINTS requires the exist POSCAR, the k density and lattice constant determine the KPOINTS.
For self-consistent calculation, users need to prepare a KPOINTS file to specify density of K-points and the method for automatic k-mesh generation.

.. code-block:: none

   > qvasp -k 0.03

Then, qvasp will ask us to input KPT-Resolved Value between K-points in reciprocal cell in units of :math:`2\pi \times 0.03 Å^{-1}`.
And in this command, the dimension of material will be considered, the vacuum direction will be set to 1. Users should note that the dimension of material may be wrong, users could revise it manually.
or ignore the dimension of material via using:

.. code-block:: none

   >qvasp -3k 0.03

Number of K-points increases when the density of k decreases. For each direction, the number is determined by

.. math::


   N=\max \left(1,\left|\vec{b}_{i}\right| / \mathrm{density}\right)

where :math:`\vec{b}_{i}` are the reciprocal lattice vectors. These values are rounded to the next integer greater than or equal to **N**. The recommend value ~0.03 (:math:`2\pi \times 0.04 Å^{-1}`) is enough for most system. This parameter is similar as the parameter KSPACING in INCAR. But the unit is different. Unit of KSPACING is :math:`Å^{-1}`, and the unit of qvasp is :math:`2\pi \times 0.03 Å^{-1}`.

The output KPOINTS file is:

.. code-block:: none

   >cat KPOINTS
   Auto
   0
   G
   16 16 16
   0.  0.  0.


Generate POTCAR
~~~~~~~~~~~~~~~~~~~
The POTCAR should be generated by the element orders in POSCAR (6th line).

.. code-block:: none

   >qvasp -pbe C

The ``RECOMMENDED_POTCAR`` tag control whether to use recommended potentials from VASP manual (Page. 195, 2018.10.29, http://cms.mpi.univie.ac.at/vasp/vasp/Recommended_PAW_potentials_DFT_calculations_using_vasp_5_2.html).

POTCAR types:

1. No extensions "_"
2. ``_d``. An extension d, treat the *d* semi core states as valence
   state.
3. ``_pv`` or ``_sv``. The extensions ``_pv`` and ``_sv`` imply that the
   *p* and *s* semi-core states are treated as valence states.
4. ``_h`` and ``_s``. An extension ``_h`` or\ ``_s`` implies that the
   potential is harder or softer than the standard potential and hence
   requires a higher or lower energy cutoff.
5. Pseudo hydrogen. ex: ``H.5``
6. ``_GW``. Used for GW calculation.

If users want to use _sv or _pv or H 1.25e pseudo potential， use command:

.. code-block:: none

   >qvasp -pbe Ca_sv H1.25

Then the POTCAR file is generated as:

.. code-block:: none

   >cat POTCAR
    PAW_PBE Ca_sv 06Sep2000                
    10.0000000000000     
    parameters from PSCTR are:
    SHA256 =  a47365830e737f14e0e6c5cf1ed81b94e081eecf0a33df105380881bc9da05d5 Ca_sv/POTCAR
    COPYR  = (c) Copyright 06Sep2000 Georg Kresse
    COPYR  = This file is part of the software VASP. Any use, copying, and all other rights are regulated by the VASP license agreement.
    COPYR  = If you do not have a valid VASP license, you may not use, copy or distribute this file.
    VRHFIN =Ca: 3s3p4s
    LEXCH  = PE
    EATOM  =  1006.0909 eV,   73.9456 Ry

    TITEL  = PAW_PBE Ca_sv 06Sep2000
    LULTRA =        F    use ultrasoft PP ?


If users want to check the type of POTCAR, use command:


.. code-block:: none

   >qvasp -cp 
   
    TITEL  = PAW_PBE Ca_sv 06Sep2000
    TITEL  = PAW_PBE H1.25 07Sep2000

The results show that the type of POTCAR is PAW_PBE, and the elements contain Ca_sv and H1.25.

Generate INCAR
~~~~~~~~~~~~~~~~~~~

qvasp can provide INCAR template for structural optimization, electronic structure calculations.

.. code-block:: none

   >qvasp -relax 
   
   The INCAR for structure optimization was generated sucessufully
   
   >cat INCAR
     SYSTEM =  qvasp 
    
    INCAR-basic
       PREC   = Normal
       ENCUT  = 400 eV
       NELMIN= 5
       LREAL = F    ## set to Auto if the valance electron number > 20
    
    #   ALGO = Fast
       EDIFF  = 1E-5
    #  ISMEAR = 0
       SIGMA = 0.2
    
    #   LDAU=.TRUE.  ## For LDA + U set
    #   LDAUTYPE=2
    #   LDAUL=-1 2   ## -1,0,1,2,3 for none,s,p,d,f
    #   LDAUU=0  3
    #   LDAUJ=0  0
    
       ISPIN = 1    ## For magnetism set
    #  MAGMOM = 3*1 -4 16*0  # Antiferromagnetism needs to be set
    
    #   NCORE = 2
    #   KPAR = 2
       NCORE = 2     ## Important to reduce calculate time, equal sqrt(cpu cores)
    
     Ionic Relaxation
       ICHARG = 2
       EDIFFG = -0.03        #force
       NSW    = 300          #max steps for IOM
       IBRION = 2
       POTIM = 0.5
       ISIF   = 2            #not change lattice:2 chang lattice:3
    #  IOPTCELL= 1 0 0  0 1 0 0 0 0   # fix or relax lattice
       PSTRESS= 0
       LCHARG = .FALSE.
       LWAVE  =  F
    
    # Use 'qvasp -p2c' to transer format (VESTA or MS)   

However, here qvasp only provide the template, users should revised them accordingly.
In addition, "qvasp -relax" means to merge "$qvasppath/exefile/INCAR/INCAR-base" and "$qvasppath/exefile/INCAR/INCAR-relax", users can customized the template.

File Format Transfer
-----------------------
qvasp provide effective tool for files format transfer, include:
1. cif files to POSCAR (support batch mode):


.. code-block:: none
   
   > qvasp -c2p C.cif Pt.cif Si.cif
   
 or for Materials Studio doping system, run:

.. code-block:: none
    
   > qvasp -c2p2 C.cif Pt.cif Si.cif

2. POSCAR to cif files:

.. code-block:: none
   
   > qvasp -p2c CONTCAR

3. OUTCAR to trajectory file (.arc, can be opened by Materials Studio).

.. code-block:: none
   
   > qvasp -out2arc

4. OUTCAR to Gauss.log, which can be opened by Gaussian Views.

.. code-block:: none
   
   > qvasp -gauss

5. density file (.castep, from the calculated result of CASTEP) to CHGCAR (VASP,can be opened by VESTA)

.. code-block:: none
   
   > qvasp -den2vasp
   
6. res file (GROMACS) to POSCAR

.. code-block:: none
   
   > qvasp -res2vasp

Insert points for Transition states
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
1. prepared the reactant and production in .vasp format, and termed as "R" and "P", then run:

.. code-block:: none
   
   > qvasp -it 4

Then the 00/POSCAR 01/POSCAR 02/POSCAR 03/POSCAR 04/POSCAR 05/POSCAR will be generated, the 00/POSCAR is the reactant and the 05/POSCAR is production, the 01/POSCAR 02/POSCAR 03/POSCAR 04/POSCAR are the inserted points.

2. If you insert the transition points using Materials Studio (Tools-->Reaction Preview), the "*.arc" file will be generated. Then rename "*.arc" with "POSCAR2" and the reactant wit "POSCAR1", then upload them on the server, run:

.. code-block:: none
   
   > qvasp -it2
   
Then the 00/POSCAR 01/POSCAR 02/POSCAR 03/POSCAR 04/POSCAR ... will be generated from "“*”.arc"

Bulid the nanotube
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
qvasp can roll any nanosheet, such as 2D materials or surface monolayer, to nanotube or nanoroll.
1. prepared the nanosheet in POSCAR format, but require that:
(1) the lattice belongs to orthorhombic crystal system (the beta angle is 90 degree) (use "qvasp -orthcell" to swith)
(2) vacuum along c direction and the long lattice direction along b direction. (use "qvasp -sc" to create supercell along b direction, use "qvasp -sw POS1" to switch (a,b,c) axis).


Here is the example for \(MoS_2\), the input is the POSCAR format.


.. code-block:: none
   
   > ls
   MoS2
   >mv MoS2 POSCAR
   >qvasp -findcell

    Read data from POSCAR file
    Find cell.vasp from POSCAR, Good Luck!
   > mv cell.vasp POSCAR
